## 2. 骨干设计与接口

### 2.1 结构约定

骨干采用轻量 CSP/C2f 结构，默认宽度和深度系数分别为 `0.5`、`0.33`，与计划书中的 YOLOv8s 规模保持一致。

- RGB 与 TIR 各自使用独立的浅层 Stem，保留模态特有的底层纹理；
- 从 P3 开始，两路前向使用同一组卷积权重，以控制参数量；
- 共享卷积后的 BatchNorm 对 RGB、TIR 分别维护参数和运行统计，避免两种模态的特征分布互相污染；
- 返回步长为 8、16、32 的 P3、P4、P5 特征，供下一阶段 QAF 融合使用；
- 当前 TIR 按数据实际存储格式读取为三通道张量，输入通道数固定为 3。

### 2.2 Python 调用方式

```python
import torch

from dualdet.models import DualStreamBackbone

model = DualStreamBackbone(width_multiple=0.5, depth_multiple=0.33)
rgb = torch.randn(2, 3, 512, 640)
tir = torch.randn(2, 3, 512, 640)
features = model(rgb, tir)

rgb_p3 = features["rgb"]["p3"]
tir_p5 = features["tir"]["p5"]
print(model.feature_channels)
```

输入必须满足以下条件：

1. 张量布局为 `[N, 3, H, W]`；
2. RGB 与 TIR 的 batch size 和空间尺寸完全一致；
3. 两个张量位于同一设备；
4. 数据加载阶段负责保证成对图像同名、同尺寸以及几何增强同步。

默认配置在 `512×640` 输入下返回以下空间尺寸：

| 特征层 | 步长 | 空间尺寸 | 默认通道数 |
|---|---:|---:|---:|
| P3 | 8 | 64×80 | 128 |
| P4 | 16 | 32×40 | 256 |
| P5 | 32 | 16×20 | 512 |

## 6. QAF 融合模块

### 6.1 计算流程

每个尺度使用一个独立的 `QAFFusionBlock`，输入同尺度 RGB 和 TIR 特征：

1. 对每种模态分别做全局均值池化和全局标准差池化；
2. 将四组质量描述拼接后输入轻量 MLP，产生两个模态 logits；
3. 通过 Softmax 得到每个样本的 `[w_rgb, w_tir]`，严格满足两者之和为 1；
4. 对两路特征加权求和，再使用 `1×1 Conv + BN + SiLU` 投影；
5. P3、P4、P5 分别预测权重，避免用单一权重描述所有尺度。

门控 MLP 的末层使用零初始化，因此首次前向的 RGB/TIR 权重均为 0.5。这样既避免未经训练的门控偏向某一模态，也让 QAF 的训练起点与固定均值融合一致。门控参数仍可通过反向传播正常学习。

### 6.2 调用接口

```python
import torch

from dualdet.models import DualStreamBackbone, MultiScaleQAF

backbone = DualStreamBackbone()
qaf = MultiScaleQAF(backbone.feature_channels)

rgb = torch.randn(2, 3, 512, 640)
tir = torch.randn(2, 3, 512, 640)
dual_features = backbone(rgb, tir)
output = qaf(dual_features["rgb"], dual_features["tir"])

fused_p3 = output.features["p3"]
p3_weights = output.modality_weights["p3"]  # shape: [N, 2]
```

`output.features` 是供检测颈部使用的融合特征字典；`output.modality_weights` 用于日志、可视化和结果分析，不应参与额外的人工规则修正。权重列顺序固定为 RGB、TIR。

## 7. 检测颈部、检测头与完整模型

### 7.1 PAN-FPN 检测颈部

`PANFPN` 接收 QAF 输出的 P3、P4、P5：

- 自顶向下路径将 P5 语义信息依次传递给 P4、P3；
- 自底向上路径将 P3 的定位信息重新聚合至 P4、P5；
- 聚合块使用项目已有的 C2f 实现，避免引入第二套基础组件；
- 输出层名称、空间尺寸和通道数保持不变，因此检测头不依赖颈部内部结构；
- 输入尺寸应能被 32 整除，P3/P4/P5 空间尺寸比例必须为 4:2:1。

### 7.2 无锚检测头

`AnchorFreeDetectHead` 在三个尺度分别使用解耦分类和回归分支：

- 类别数默认为 5，与课程数据类别一致；
- 不设置独立 objectness 分支，与 YOLOv8 风格保持一致；
- 每个位置输出 `4 × reg_max` 个框分布 logits，默认 `reg_max=16`，因此回归通道数为 64；
- 每个位置输出 5 个类别 logits；
- 分类输出层按 1% 前景先验初始化，减少训练初期大量背景位置造成的梯度冲击；
- 分支使用深度可分离卷积控制参数量。

三个尺度的步长固定为 P3=8、P4=16、P5=32。检测头当前输出原始 logits，不在模型层内执行标签分配、DFL 解码、损失计算、置信度过滤或 NMS。

### 7.3 完整模型调用接口

```python
import torch

from dualdet.models import QAFDetector

model = QAFDetector(num_classes=5, reg_max=16)
rgb = torch.randn(2, 3, 512, 640)
tir = torch.randn(2, 3, 512, 640)
output = model(rgb, tir)

p3_boxes = output.predictions["p3"].box_distribution
p3_classes = output.predictions["p3"].class_logits
p3_weights = output.modality_weights["p3"]
print(model.parameter_breakdown())
```

`DetectorOutput` 包含：

- `predictions`：以 `p3`、`p4`、`p5` 为键的原始预测；
- `box_distribution`：形状 `[N, 4×reg_max, H, W]`；
- `class_logits`：形状 `[N, num_classes, H, W]`；
- `modality_weights`：QAF 的 `[RGB, TIR]` 权重，仅用于记录与分析。

默认 `512×640` 输入的输出契约如下：

| 特征层 | 框分布输出 | 分类输出 |
|---|---|---|
| P3 | `[N, 64, 64, 80]` | `[N, 5, 64, 80]` |
| P4 | `[N, 64, 32, 40]` | `[N, 5, 32, 40]` |
| P5 | `[N, 64, 16, 20]` | `[N, 5, 16, 20]` |

## 8. 固定融合开关与主模型配置

### 8.1 E1/E2 公平对比设计

`MultiScaleQAF` 和 `QAFDetector` 支持两种融合模式：

- `fixed`：每个样本、每个尺度强制使用 `[0.5, 0.5]`；
- `qaf`：根据 RGB/TIR 特征的均值和标准差动态预测权重。

两种模式实例化完全相同的 QAF 门控和投影层，因此模型结构和总参数量均为 9,178,173。固定模式只跳过门控计算结果，不删除门控参数，确保 E1/E2 的参数规模一致。E1 中门控参数不会获得梯度，这是固定融合的预期行为；其余 backbone、投影、neck 和 head 正常训练。

可以在构建模型时指定模式：

```python
from dualdet.models import QAFDetector

e1_model = QAFDetector(fusion_mode="fixed")
e2_model = QAFDetector(fusion_mode="qaf")
```

也可以在调试时切换已有融合模块：

```python
e2_model.qaf.set_fusion_mode("fixed")
e2_model.qaf.set_fusion_mode("qaf")
```

正式实验不建议在同一次运行中切换模式；每次训练应从对应配置单独创建模型，并在日志中记录配置文件路径。

### 8.2 配置文件

| 实验 | 配置文件 | 唯一实验变量 |
|---|---|---|
| E1 | `configs/model_e1_fixed.json` | `fusion_mode=fixed` |
| E2 | `configs/model_e2_qaf.json` | `fusion_mode=qaf` |

E2 是当前主模型配置。两份配置固定以下内容：

- 类别顺序：`car`、`truck`、`bus`、`van`、`freight_car`；
- 输入尺寸：512×640；
- 宽度系数：0.5；
- 深度系数：0.33；
- 最大通道数：512；
- PAN-FPN C2f 重复数：1；
- DFL `reg_max`：16；
- 类别数由 `class_names` 自动计算为 5。

除 `experiment_name` 和 `fusion_mode` 外，两份配置必须完全相同。修改其中一份结构设置时必须同步修改另一份，否则公平性测试会失败。

加载配置并构建模型：

```python
from dualdet.config import ModelConfig

config = ModelConfig.from_json("configs/model_e2_qaf.json")
model = config.build_model()
print(config.num_classes)
```
