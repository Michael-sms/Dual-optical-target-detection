# 成员 3：P2 小目标检测头与指标分析工具说明

## 1. 已完成内容

本阶段完成不依赖最终 checkpoint 的成员 3 交付项：

- 可选 P2 Head 结构：在原 P3/P4/P5 检测路径外，新增 stride 4 的 P2 检测输出；
- E3 模型配置：新增 `configs/model_e3_qaf_p2.json`，用于后续与 E2 做受控对比；
- 指标分析工具：新增轻量 COCO 预测分析脚本，可输出每类 precision/recall、小/中/大目标召回和混淆矩阵；
- 测试用例：覆盖 P2 neck、四尺度 detection head、P2 detector 和 E2/E3 配置控制关系。

正式 E1-E3 数值对比需要等待成员 2 产出可复现 checkpoint 和验证集预测结果。

## 2. P2 Head 设计

### 2.1 接入方式

默认模型保持原有三尺度输出：

```python
QAFDetector(use_p2_head=False)
```

开启 E3 时使用：

```python
QAFDetector(use_p2_head=True)
```

P2 特征来自双流骨干的 stride 4 浅层 stem 输出。考虑到计划书约定 QAF 只在 P3/P4/P5 尺度进行，P2 使用固定 `0.5 RGB + 0.5 TIR` 融合，避免在 E3 中同时引入新的质量门控变量。随后 P2 通过 `P2PANFPN` 接入检测颈部，并由同一 `AnchorFreeDetectHead` 产生 `p2/p3/p4/p5` 四尺度预测。

### 2.2 代码位置

- `dualdet/models/backbone.py`
  - 新增 `forward_with_p2()`；
  - 新增 `stem_feature_channels` 和 `feature_channels_with_p2`。
- `dualdet/models/neck.py`
  - 新增 `P2PANFPN`。
- `dualdet/models/head.py`
  - `AnchorFreeDetectHead` 支持可配置 `feature_names`。
- `dualdet/models/detector.py`
  - 新增 `use_p2_head` 开关；
  - E3 启用 P2 固定融合和四尺度预测。

### 2.3 受控对比口径

- E1：`fusion_mode="fixed"`，`use_p2_head=false`；
- E2：`fusion_mode="qaf"`，`use_p2_head=false`；
- E3：`fusion_mode="qaf"`，`use_p2_head=true`。

因此 E2/E3 的唯一结构变量是 P2 Head，后续实验表中应保持数据划分、输入尺寸、随机种子、训练轮数、优化器和预训练权重一致。

## 3. 指标分析工具

### 3.1 命令

```powershell
python scripts/analyze_predictions.py `
  --annotations data/val/val.json `
  --predictions outputs/e3_val_predictions.json `
  --markdown-output outputs/analysis/e3_analysis.md `
  --json-output outputs/analysis/e3_analysis.json
```

### 3.2 输出内容

工具会生成：

- 每类 `GT / Pred / TP / FP / FN / Precision / Recall`；
- `small / medium / large` 三个 COCO 面积分桶的召回；
- 基于最佳 IoU 匹配的混淆矩阵；
- 总体 precision 和 recall。

该工具不替代官方 COCO AP。报告中的 AP/AP50/AP75 应以成员 4 的 COCO 评估或官方网页结果为准；本工具用于成员 3 的小目标分析、每类错误定位和定性案例筛选。

## 5. 消融分析文字

本地验证结果显示，E1 固定双流融合相对 E0 单 RGB 基线有明显提升，AP 从 0.328839 提升到 0.480500，说明 RGB 与 TIR 的互补信息对无人机双光目标检测是有效的。E1 同时也是线上最优单模型，线上 mAP 为 0.246919，说明固定融合虽然结构简单，但泛化更稳定，可作为最终 ensemble 的主力成员。

E2 在本地验证集上达到最高 AP 0.484318，比 E1 高 0.003819，AP50、AP75 和 AP_small 也略优，说明质量感知门控融合能够利用场景质量差异，增强 RGB/TIR 的动态互补。但根据线上结果，E2 没有超过 E1，推测原因是验证集和测试集存在域偏移，QAF 门控在本地验证集上学到的模态偏好在线上场景中不完全稳定。因此 E2 更适合作为补充模型参与融合，而不是单独替代 E1。

E3 在 E2 基础上加入 P2 Head 后，本地总体 AP 为 0.483146，低于 E2 的 0.484318，说明 P2 单模型没有带来总体最优；同时参数量从 9.18M 增至 9.44M，模型复杂度略有增加。不过 E3 的 AP_small 达到 0.166479，高于 E2 的 0.160904，说明 P2 高分辨率分支确实增强了小目标候选的召回能力。因此 E3 不作为最终单模型，但保留为 ensemble 成员，为密集小目标和远距离车辆提供互补预测。

三模型融合最终线上 mAP 为 0.271375，相对 E1 线上最优单模型 0.246919 提升 0.024456，约 2.45 个百分点。该结果说明 E1 的稳定泛化、E2 的质量感知补充和 E3 的小目标补偿具有互补性，融合后可以覆盖更多测试场景。

## 6. 最终融合提升

| 项目 | mAP |
|---|---:|
| E1 线上最优单模型 | 0.246919 |
| 三模型最终融合 | 0.271375 |
| 绝对提升 | 0.024456 |
| 百分点提升 | 约 2.45 个百分点 |

最终融合使用 `result_e1nms08_e2_e3_nms70_s002_compact.json`。从预测分布看，融合结果共有 57241 个预测框，相比 E2 单模型的 30319 个预测框覆盖更充分；少数类候选也更丰富，其中 truck、van、freight_car 的预测数量均明显增加，有利于缓解测试集中的长尾类别漏检。

## 7. 小目标与 P2 Head 结论

P2 Head 的核心价值不在于直接提高单模型总体 AP，而在于提供小目标补充。E3 的 AP_small 比 E2 高 0.005575，且在若干测试图上明显增加小目标候选。例如候选案例 `00034` 中，E2 小目标预测数为 62，E3 增至 142；案例 `00680` 中，E2 小目标预测数为 87，E3 增至 155。这说明 P2 分支能够利用 stride 4 的浅层空间细节，补回部分远距离小车和密集车辆。

但是 P2 也会带来额外低分候选和类别噪声，尤其在密集区域容易增加重复框或少数类误判。因此最终策略不是让 E3 单独提交，而是将其作为 ensemble 成员，通过融合和 NMS 保留有价值的小目标补充，同时抑制不稳定预测。

## 8. 典型案例图与错误分析

已根据 E1、E2、E3 和最终融合预测生成候选案例文件：

```text
output/outputs/member3_case_candidates.json
```

建议整理以下三类可视化案例，分别从测试集 RGB/TIR 图像中取对应 `image_id` 画框对比：

| 案例类型 | 推荐 image_id | 观察重点 |
|---|---|---|
| 密集小目标 | 662, 656, 334, 663, 680 | 车辆密集且框尺寸小，容易出现漏检和 NMS 抑制 |
| P2 补充小目标 | 34, 680, 335, 708, 661 | E3/P2 相比 E2 增加小目标候选，可展示 P2 的补偿作用 |
| 少数类补充 | 715, 521, 455, 469, 462 | truck、van、freight_car 等少数类在融合后候选更多 |

主要错误类型如下：

1. 密集小车漏检：无人机俯视下车辆尺寸小、间距近，低分小框容易被 NMS 或置信度阈值过滤。
2. 长尾类别混淆：truck、van、freight_car 外观相近且样本少，模型容易在厢式货车、卡车和大货车之间混淆。
3. P2 额外候选噪声：P2 提升小目标候选数量，但也可能带来重复框和低置信度误检，需要依赖 ensemble 后处理过滤。
4. 域偏移影响：E2 本地验证高于 E1，但线上不如 E1，说明测试集光照、场景或热红外质量分布与验证集不同，动态门控存在泛化波动。
