## 2. P2 Head 设计

### 2.1 接入方式

默认模型保持原有三尺度输出：

```python
QAFDetector(use_p2_head=False)
```

开启 E3 时使用：

```python
QAFDetector(use_p2_head=True)
```

P2 特征来自双流骨干的 stride 4 浅层 stem 输出。考虑到计划书约定 QAF 只在 P3/P4/P5 尺度进行，P2 使用固定 `0.5 RGB + 0.5 TIR` 融合，避免在 E3 中同时引入新的质量门控变量。随后 P2 通过 `P2PANFPN` 接入检测颈部，并由同一 `AnchorFreeDetectHead` 产生 `p2/p3/p4/p5` 四尺度预测。

### 2.2 代码位置

- `dualdet/models/backbone.py`
  - 新增 `forward_with_p2()`；
  - 新增 `stem_feature_channels` 和 `feature_channels_with_p2`。
- `dualdet/models/neck.py`
  - 新增 `P2PANFPN`。
- `dualdet/models/head.py`
  - `AnchorFreeDetectHead` 支持可配置 `feature_names`。
- `dualdet/models/detector.py`
  - 新增 `use_p2_head` 开关；
  - E3 启用 P2 固定融合和四尺度预测。

### 2.3 受控对比口径

- E1：`fusion_mode="fixed"`，`use_p2_head=false`；
- E2：`fusion_mode="qaf"`，`use_p2_head=false`；
- E3：`fusion_mode="qaf"`，`use_p2_head=true`。

因此 E2/E3 的唯一结构变量是 P2 Head，后续实验表中应保持数据划分、输入尺寸、随机种子、训练轮数、优化器和预训练权重一致。

## 3. 指标分析工具

### 3.1 命令

```powershell
python scripts/analyze_predictions.py `
  --annotations data/val/val.json `
  --predictions outputs/e3_val_predictions.json `
  --markdown-output outputs/analysis/e3_analysis.md `
  --json-output outputs/analysis/e3_analysis.json
```

### 3.2 输出内容

工具会生成：

- 每类 `GT / Pred / TP / FP / FN / Precision / Recall`；
- `small / medium / large` 三个 COCO 面积分桶的召回；
- 基于最佳 IoU 匹配的混淆矩阵；
- 总体 precision 和 recall。

该工具不替代官方 COCO AP。报告中的 AP/AP50/AP75 应以成员 4 的 COCO 评估或官方网页结果为准；本工具用于成员 3 的小目标分析、每类错误定位和定性案例筛选。
