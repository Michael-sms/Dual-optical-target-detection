# 成员 3 消融分析与 P2 Head 结论

## 1. 本地验证消融结果

| 实验 | 方法 | P2 | 本地 AP | AP50 | AP75 | AP_small | 参数量 | 结论 |
|---|---|---:|---:|---:|---:|---:|---:|---|
| E0 | RGB baseline | 否 | 0.328839 | 0.569807 | 0.340511 | 0.091623 | 8.70M | 单 RGB 基线，验证双光融合必要性 |
| E1 | 固定双流融合 | 否 | 0.480500 | 0.681237 | 0.577963 | 0.157990 | 9.09M | 本地强基线，也是线上最优单模型 |
| E2 | QAF 质量感知融合 | 否 | 0.484318 | 0.685781 | 0.584429 | 0.160904 | 9.18M | 本地验证最高，说明 QAF 有效 |
| E3 | QAF + P2 Head | 是 | 0.483146 | 0.681714 | 0.584813 | 0.166479 | 9.44M | 小目标 AP 最高，但总体 AP 未超过 E2 |

## 2. 消融分析文字

E1 固定双流融合相对 E0 单 RGB 基线提升显著，本地 AP 从 0.328839 提升到 0.480500，说明 RGB 与 TIR 的互补信息能够有效提升无人机双光目标检测性能。E1 同时是线上最优单模型，线上 mAP 为 0.246919，说明固定融合虽然简单，但泛化稳定。

E2 在本地验证集上达到最高 AP 0.484318，比 E1 高 0.003819，AP50、AP75 和 AP_small 均略优，说明质量感知门控融合能够动态利用不同模态的信息。但 E2 线上结果不如 E1，说明验证集和测试集存在域偏移，QAF 的模态权重在测试场景中可能不如固定融合稳定。因此 E2 适合作为融合成员，而不是直接替代 E1。

E3 加入 P2 Head 后，本地总体 AP 为 0.483146，低于 E2 的 0.484318，说明 P2 单模型没有超过 E2；同时参数量从 9.18M 增加到 9.44M。不过 E3 的 AP_small 达到 0.166479，高于 E2 的 0.160904，说明 P2 高分辨率分支确实带来了小目标补偿。因此 E3 不作为最终单模型，但保留为 ensemble 成员。

## 3. 最终融合提升

| 项目 | mAP |
|---|---:|
| E1 线上最优单模型 | 0.246919 |
| 三模型阶段性融合 | 0.271375 |
| 四模型阶段性融合 | 0.278432 |
| 预训练 TIR YOLOv8m 单模型 | 0.327517 |
| 五模型最终融合 | 0.357833 |
| 相对 TIR 单模型绝对提升 | 0.030316 |

最终采用 TIR 主导的五模型融合。预训练 TIR YOLOv8m 提供高精度主预测，原四模型以较低权重补充召回。最终融合预测文件为：

```text
result_final.json
```

最终文件使用 `NMS IoU=0.68`，TIR 权重 `1.0`、阈值 `0.002`，原四模型权重 `0.25`、阈值 `0.05`，共输出 55094 个预测框。加权融合兼顾 TIR 的定位精度与自研双流模型的召回能力。

## 4. P2 Head 小目标结论

P2 Head 没有带来线上最优单模型，也没有在本地总体 AP 上超过 E2，但它提高了小目标指标。E3 的 AP_small 比 E2 高 0.005575，说明 stride 4 的 P2 分支可以利用更高分辨率的浅层位置细节，补充远距离车辆和密集小车。

典型候选：

|---:|---:|---:|---|
| 34 | 62 | 142 | P2 明显补充小目标候选 |
| 680 | 87 | 155 | P2 增加密集场景候选 |

| 708 | 55 | 108 | P2 增加小目标召回候选 |
| 661 | 99 | 149 | P2 在密集小车场景有补充 |

因此，P2 Head 的定位是 ensemble 补充成员，而不是最终单模型。

## 6. 错误分析

1. 密集小车漏检：车辆尺度小且相互靠近，低分小框容易被 NMS 或阈值过滤。
2. 少数类混淆：truck、van、freight_car 外观相近，且样本量远少于 car，容易相互误判。
3. P2 噪声：P2 能补充小目标候选，但也会产生更多低置信度框和重复框，需要融合 NMS 过滤。
4. 域偏移：E2 本地验证高于 E1，但线上不如 E1，说明测试集光照、天气、热红外质量和验证集不同，动态门控存在泛化波动。
