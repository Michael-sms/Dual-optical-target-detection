# 无人机视角下双光目标检测项目计划书

> 项目代号：DualSight-Det  
> 团队规模：4 人  
> 计划版本：v1.2（2026-06-20，七日精简版）  
> 项目周期：2026-06-20 至 2026-06-26（第 6 天冻结代码，第 7 天完成正式提交）  
> 数据范围：仅使用课程提供的 RGB-T 数据；允许使用并如实报备开源预训练权重

## 1. 项目概述

本项目面向无人机视角下的 RGB（可见光）与 TIR（热红外）成对图像，检测小轿车、卡车、公交车、厢式货车/面包车和大货车五类目标。无人机图像中目标尺度偏小、密集遮挡多，且可见光模态容易受夜间、逆光和天气影响；TIR 在低照度下更稳定，但纹理和类别细节较弱。因此，本项目将以轻量级单阶段检测器为基线，构建“**质量感知的双流跨模态融合 + 小目标增强**”模型，在提高 COCO AP 的同时控制模型参数量。

最终交付包括：测试集检测结果 JSON、可复现实验代码及 README、3 页以内实验报告、成员分工与贡献表，以及预训练模型来源和 MD5 记录。

### 2.1 检测类别

| category_id | 类别名称 | 中文说明 |
|---:|---|---|
| 1 | car | 小轿车 |
| 2 | truck | 卡车 |
| 3 | bus | 公交车 |
| 4 | van | 厢式货车/面包车 |
| 5 | freight_car | 大货车 |

### 2.2 官方评估规则

- 主排名指标为 COCO 风格 **AP@[0.50:0.95]**，同时观察 AP50 和 AP75。
- 若 AP 相同，模型参数大小将用于进一步排名；因此不能只追求大模型精度。
- 测试结果为单个 JSON List，每个预测框格式如下：

```json
{
  "image_id": 1,
  "category_id": 1,
  "bbox": [154.0, 158.0, 25.0, 63.0],
  "score": 0.95
}
```

- `image_id` 为文件名去掉后缀后转为整数，例如 `00001.jpg -> 1`。
- `bbox` 必须为 `[x, y, width, height]`，坐标原点为左上角；推理程序内部若使用 `xyxy`，导出时必须转换。
- 坐标建议保留 1 位小数，置信度必须是合法浮点数；不得输出 NaN、Infinity、负宽高或越界框。
- 网页提交时同时填写模型大小；提交后正常情况下约 30 秒可见结果，超过 5 分钟后台会停止评测。

### 2.3 数据与合规约束

- 禁止未经批准使用外部数据集。
- 可使用开源预训练模型；最终文档必须记录下载地址、许可证、文件名和实际 MD5。
- 任何增强都只作用于课程训练数据；成对图像的几何增强必须使用同一随机参数。

### 3.2 类别分布

| 类别 | 训练框数 | 训练占比 | 验证框数 |
|---|---:|---:|---:|
| car | 270,348 | 85.44% | 20,588 |
| truck | 15,833 | 5.00% | 1,470 |
| bus | 11,334 | 3.58% | 789 |
| van | 7,701 | 2.43% | 725 |
| freight_car | 11,193 | 3.54% | 918 |

按 COCO 面积阈值粗略统计，训练集中小目标（面积小于 `32²`）有 43,951 个，约占 13.9%；中等面积目标占多数。结合无人机俯视、密集车辆与测试域变化，项目的主要困难为：

1. **长尾类别**：car 数量远高于其他四类，容易压制少数类分类梯度。
2. **小目标与密集目标**：下采样导致细节丢失，NMS 还可能误删相邻车辆。
3. **模态质量动态变化**：夜间 RGB 退化，而白天或热背景下 TIR 不一定占优，固定比例融合不合理。
4. **跨模态轻微错位**：成对传感器存在视差或标定误差，直接逐像素相加可能产生重影特征。
5. **测试域偏移**：测试集与训练集采集时间不同，天气、光照和场景分布存在变化。

## 4. 技术路线

### 4.1 基线选择

首选 **YOLOv8s** 作为工程基线：训练和导出流程成熟，速度与模型大小适中，便于增加双流输入和开展消融。若计算资源不足，先用 YOLOv8n 验证流程；最终模型在 YOLOv8n/s 的验证 AP 与模型大小之间选择。

必须建立以下可比较基线：

| 编号 | 输入/方法 | 目的 |
|---|---|---|
| B0 | RGB 单模态 YOLOv8s | 测量可见光上限与低照度短板 |
| B1（可选） | TIR 单模态 YOLOv8s | 仅在首日流程顺畅时补充，测量红外独立贡献 |
| B3 | 双流骨干 + 固定融合 | 排除“仅增加参数量”带来的增益 |
| P | 本项目创新模型 | 与 B3 同训练策略做公平比较 |

### 4.2 创新方案：QAF-SOD 双光检测器

创新模型命名为 **QAF-SOD（Quality-Aware Fusion for Small-Object Detection）**。七日周期内只实现两个核心设计：QAF 为必须完成项，P2 Head 为资源允许时加入的增强项；不再单独开发一致性训练等高成本模块。

#### 创新点一：模态质量感知门控融合（QAF）

RGB 与 TIR 分别使用轻量双流骨干提取多尺度特征。为控制模型大小，两个分支共享骨干的模块结构，并在中高层使用共享卷积权重、独立归一化参数。对第 `l` 层特征 `F_rgb^l`、`F_tir^l`：

1. 分别进行全局平均池化和全局标准差池化，得到亮度、对比度与响应强度的质量描述；
2. 通过轻量 MLP 预测每个尺度的模态权重 `w_rgb^l` 与 `w_tir^l`，经 Softmax 约束两者和为 1；
3. 使用轻量卷积整合加权后的双模态特征：

```text
F_fuse^l = Conv(w_rgb^l * F_rgb^l + w_tir^l * F_tir^l)
```

质量门控让模型根据场景动态选择模态。融合安排在 P3、P4、P5 尺度，避免原图级逐像素融合对轻微错位过度敏感。七日版本不增加独立差异分支，以减少开发与消融成本。

#### 创新点二：高分辨率小目标检测头（P2 Head）

在常规 P3-P5 检测头基础上增加步长为 4 的 P2 检测头，将浅层边缘与位置细节送入检测颈部。P2 采用深度可分离卷积并限制通道数，以较小参数代价改善小目标召回。训练与验证额外报告小目标 AP、按目标面积分桶的 Recall，判断增益是否确实来自小目标。

### 4.3 类别不均衡与密集目标策略

- 分类损失沿用框架默认设置；只有当 E0 明确出现严重少数类失衡时，才单独尝试 Focal Loss，且最多进行一次筛选实验。
- 默认不改采样策略；若后续采用少数类图像加权采样，必须记录为额外变量，禁止简单复制单框或破坏场景分布。
- 增强配置保守使用 Mosaic、MixUp 和随机尺度；双模态几何操作完全同步，并在训练末段关闭强增强。
- 密集场景沿用标准 NMS，仅在验证集小范围调整置信度与 NMS IoU，不新增 Soft-NMS 实验，也不依据测试榜单反复过拟合。
- 每类分别检查 AP 和混淆矩阵，尤其关注 truck、bus、van、freight_car 之间的混淆。

### 5.1 推荐目录结构

```text
Dual-optical target detection/

```

### 5.2 数据管线验收项

- 按 COCO `images.id` 建立样本索引，不依赖目录枚举顺序。
- 每次读取前断言 RGB、TIR 同名且尺寸一致；缺失配对直接报错，不静默跳过。
- 对 TIR 明确采用三通道读取或单通道后复制，训练和推理保持一致。
- 同步增强后，检查框坐标合法并过滤过小/无效框。
- 用 16 个样本画出 RGB、TIR 与框的可视化网格，由两名成员交叉检查。
- 固定随机种子，保存完整配置、代码提交号、硬件、训练时长和最佳 epoch。

### 5.3 推荐初始训练配置

以下为起始值，最终以显存和验证集结果为准：

| 项目 | 初始设置 |
|---|---|
| 输入尺寸 | 固定 640×512，不开展多尺寸对比 |
| 优化器 | AdamW 或框架默认优化器，统一基线设置 |
| 训练轮数 | 筛选实验 30-50 epochs，最终模型 60-80 epochs；早停 patience 10 |
| 学习率 | 按有效 batch size 线性缩放，使用 warmup + cosine decay |
| 精度 | AMP 混合精度 |
| 模型选择 | 以验证 AP@[.50:.95] 最佳 checkpoint 为准 |
| 日志 | 总损失、各子损失、AP/AP50/AP75、每类 AP、参数量、FLOPs、显存与推理时延 |

为保证实验公平，所有核心实验使用相同数据划分、随机种子、输入尺寸、预训练权重和训练轮数。七日周期内统一采用单随机种子；若剩余算力充足，只对固定双流基线和最终模型各补 1 次复验。复验不是提交前置条件，但必须在报告中如实说明随机种子与算力限制。

## 6. 实验矩阵与决策规则

### 6.1 核心消融（最小验证集）

| 实验 | 固定双流 | QAF | P2 | 优先级 | 要回答的问题 |
|---|:---:|:---:|:---:|---|---|
| E0 | - | - | - | 必做 | RGB 单模态基线达到什么水平？ |
| E1 | ✓ | - | - | 必做 | 双模态固定融合本身是否有效？ |
| E2 | ✓ | ✓ | - | 必做 | 质量感知融合是否优于固定融合？ |
| E3 | ✓ | ✓ | ✓ | 条件执行 | P2 是否以可接受的模型开销改善小目标？ |

核心结论只依赖 E0-E2，避免为每个内部算子继续拆分实验。E3 仅在第 4 天中午前 E2 已获得完整结果且训练资源充足时启动；若资源不足，原计划直接采用 E2。实际最终提交以最终方案冻结记录为准。

每个必做实验记录总体 AP、AP50、AP75、参数量和权重文件大小；仅 E1-E3 补充五类 AP 与小目标 AP/Recall。FLOPs 和单张推理时延只统计最终候选，不要求每个消融重复测量。若 E2 相对 E1 无明确增益，则回退到 E1 并在报告中如实分析；若 E3 增益小于 0.3 个百分点或开销明显增加，则不纳入最终模型。

### 6.2 简化分析

不再单独构造六类退化验证集，也不开展 MDC、Soft-NMS、TTA、输入尺寸网格搜索等扩展实验。报告只选取 20-30 张代表性验证图进行定性对比，并分析夜间、密集小目标和易混类别三类典型案例；该分析不得占用新的完整训练轮次。

### 6.3 最终模型选择

采用双目标决策：

1. 首先选择验证 AP 较高、训练过程正常且可复现的模型；
2. 若候选模型 AP 差距小于 0.3 个百分点，优先选择参数量和权重文件更小者；
3. 置信度与 NMS 阈值只在验证集小范围确定，不启用 TTA；
4. 测试网页用于最终确认，不进行无记录的高频试探。

### 6.4 实际执行与最终冻结记录

在完成原定 E0-E3 后，项目利用新增 RTX 5090 算力补训 E1 fixed fusion 的 v8m 尺度版本。v8m E1 本地 AP 为 `0.483802`，单模型线上 mAP 为 `0.244951`，未超过 v8s E1 的 `0.246919`，因此不替换最优单模型；但其预测与原有模型具有互补性，加入结果级融合后带来稳定增益。

第一阶段冻结方案为原四模型融合，随后依据实践允许使用开源预训练模型的要求，补充了 COCO 预训练 YOLOv8m 单模态实验。RGB YOLOv8m 线上 mAP 仅为 `0.090413`，未被采用；TIR YOLOv8m 单模型达到 `0.327517`，最终与原四模型进行加权结果级融合。

最终冻结方案如下：

- 融合成员：预训练 TIR YOLOv8m + E1 fixed fusion v8s + E1 fixed fusion v8m + E2 QAF + E3 QAF+P2；
- 最终文件：`outputs/result_final.json`；
- 后处理：TIR 权重 `1.0`、阈值 `0.002`，原四模型权重 `0.25`、阈值 `0.05`，按类别执行 `NMS IoU=0.68`；
- 参数量填写：`78.75M`；
- 最终线上 mAP：`0.357833`。

原三模型融合 `0.271375` 和原四模型融合 `0.278432` 均作为阶段性结果保留，不再作为最终提交口径。

## 10. 结果导出与提交前检查

### 10.1 JSON 自动校验

推理脚本生成 `result.json` 后必须自动检查：

- 根对象是 List，每个元素仅含所需字段；
- `image_id` 属于 1-1000 的测试图像 ID 集合，不把前导零字符串直接写入；
- `category_id` 只能为 1-5；
- `bbox` 长度为 4，`x,y >= 0`，`width,height > 0`，并裁剪到 640×512 范围；
- `score` 在 `[0,1]` 内，所有数字均为有限值；
- 无明显重复框；结果文件可被标准 JSON 解析器重新读取；
- 从 20 张随机测试图生成可视化，人工检查 RGB 和 TIR 场景中的框与类别。

## 11. 项目成功标准

1. **正确性**：本地 COCO 评估和网页评测均能正常运行，提交格式零错误。
2. **有效性**：完整模型相对参数量可比的固定双流基线获得可复验的 AP 提升，并通过消融说明提升来源。
3. **创新性**：质量感知融合相对固定融合有定量对比；若采用 P2 Head，则同时给出小目标指标与模型开销。
4. **轻量性**：同时报告 AP、参数量、模型文件大小和时延；当精度接近时选择更小模型。
5. **可复现性**：新环境按 README 能完成数据检查、训练、验证、推理和 JSON 导出。
6. **协作性**：四名成员职责和产出可追踪，代码至少一人交叉评审，贡献度有客观依据。
