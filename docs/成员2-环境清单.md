# 成员 2 环境清单与工程说明

本文档面向项目复现与协作交接，说明当前仓库结构、成员 2 负责的数据与训练工程，以及 E0–E3 四组实验的运行方式与产出约定。

## 1. 项目概览

本项目为天津大学计算机视觉实践课程的双光（RGB + TIR）目标检测任务：在无人机视角下，利用可见光与热红外成对图像检测 5 类车辆目标（car / truck / bus / van / freight_car）。

按计划书控制变量设计，仓库内实现了四组消融实验：

| 实验 ID | 名称 | 模型要点 | 特征层 |
|---|---|---|---|
| E0 | RGB Baseline | 仅 RGB 单流骨干，TIR 输入被忽略 | P3 / P4 / P5 |
| E1 | Fixed Fusion | 双流骨干 + 固定权重多尺度融合 | P3 / P4 / P5 |
| E2 | QAF | 双流骨干 + 质量感知融合（QAF，成员 1 实现） | P3 / P4 / P5 |
| E3 | QAF + P2 | E2 基础上增加轻量 P2 Head（成员 3 提供模块） | P2 / P3 / P4 / P5 |

成员 2 的主责是：**数据管线、检测损失、训练循环、COCO 验证评估、实验复现环境与训练产物管理**。模型骨干 / QAF / P2 等结构由成员 1、3 提供接口，成员 2 负责将其接入可训练、可评估、可复现的完整流水线。

### 2.1 `dualdet` 包模块说明

#### `dualdet/datasets/` — 数据层（成员 2）

| 文件 | 作用 |
|---|---|
| `coco.py` | 解析 COCO JSON，构建 `COCOAnnotationIndex`；bbox 统一为 xyxy 像素坐标 |
| `rgb_t.py` | `RGBTDataset`：按文件名配对加载 RGB/TIR；过滤过小框；返回 tensor + boxes + labels |
| `transforms.py` | `SyncAugmentPipeline`：RGB/TIR/框 **共享** resize（640×512）与水平翻转，防止模态错位 |

#### `dualdet/losses/` — 损失与标签分配（成员 2）

| 文件 | 作用 |
|---|---|
| `assigner.py` | `TaskAlignedAssigner`：Task-Aligned 正样本分配 |
| `bbox.py` | CIoU、DFL 解码、网格点生成 |
| `detection.py` | `DetectionLoss`：box（CIoU）+ cls（BCE）+ dfl，支持 P3–P5 或 P2–P5 多尺度 |

#### `dualdet/models/` — 模型结构（成员 1 / 3，成员 2 通过 `build.py` 接入训练）

| 文件 | 作用 |
|---|---|
| `backbone.py` | 双流骨干 `DualStreamBackbone` |
| `rgb_baseline.py` | E0 单流 RGB 骨干 |
| `fusion.py` | E1 固定融合 `FixedMultiScaleFusion` |
| `qaf.py` | E2/E3 质量感知融合 `MultiScaleQAF` |
| `neck.py` | `PANFPN` / `PANFPNWithP2` |
| `head.py` | 无锚检测头 `AnchorFreeDetectHead` |
| `detector.py` | `DetectorOutput(predictions, modality_weights)` 统一输出 |
| `build.py` | `build_detector("e0"\|"e1"\|"e2"\|"e3")` 实验工厂 |

#### `dualdet/utils/` — 训练与评估工具（成员 2 为主）

| 文件 | 作用 |
|---|---|
| `config.py` | YAML 配置加载 |
| `collate.py` | 变长 bbox 的 batch 拼接 |
| `amp.py` | AMP 自动混合精度上下文与 GradScaler |
| `seed.py` | 全局随机种子 |
| `checkpoint.py` | 保存 / 加载 `best.pt`、`last.pt` |
| `coco_eval.py` | 每 epoch 验证集 COCO AP / AP50 / AP75 / 小中大目标 AP |
| `postprocess.py` | 解码、NMS（供评估与成员 4 推理复用） |
| `run_metadata.py` | `run_metadata.json`、`training_summary.json` 生成 |
| `inference.py` / `submission.py` | 测试集推理与 JSON 导出（成员 4 扩展） |

### 2.2 训练数据流（成员 2 串联的全链路）

```
COCO JSON + RGB/TIR 图像
        ↓
  RGBTDataset + SyncAugmentPipeline
        ↓
  detection_collate → batch {rgb, tir, targets}
        ↓
  build_detector(experiment) → DetectorOutput.predictions
        ↓
  DetectionLoss → total / box / cls / dfl
        ↓
  AdamW + Cosine LR + AMP 反传
        ↓
  每 epoch：验证 loss + evaluate_coco_metrics(AP)
        ↓
  按 val.ap 保存 best.pt；早停；写入 history / summary
```

### 5.4 四组实验结果摘要

训练配置统一：seed=42，batch=8，lr=1e-3，warmup=3 epoch，早停 patience=10（最少训练 20 epoch 后生效），**best checkpoint 按 val.ap 选取**。

| 实验 | 实际 epoch | best epoch | best val AP | best val loss | 参数量 | 训练时长 |
|---|---:|---:|---:|---:|---:|---:|
| E0 RGB Baseline | 40 | 30 | 0.329 | 3.841 | 8.70M | ~3.5 h |
| E1 Fixed Fusion | 34 | 24 | 0.481 | 2.568 | 9.09M | ~2.4 h |
| E2 QAF | 42 | 32 | **0.484** | **2.538** | 9.18M | ~3.0 h |
| E3 QAF + P2 | 31 | 21 | 0.474 | 2.988 | 9.23M | ~2.4 h |

初步结论（供成员 3 进一步分析）：

- 双流 + 融合相对 E0 提升显著（AP +15 pp 左右）
- E2 QAF 略优于 E1 Fixed Fusion，为当前最佳验证 AP
- E3 P2 Head 在本配置下未超过 E2，可能需成员 3 做每类 AP / 小目标专项分析

## 7. 复现命令

### 7.1 数据与增强检查

```bash
# 数据完整性检查
python scripts/check_data.py

# 同步增强可视化（计划书 §5.2 要求 16 张）
python scripts/visualize_augment.py --split train --num-samples 16
```

### 7.2 训练

```bash
# 一条命令顺序训练 E0–E3（推荐）
bash scripts/train_all.sh

# 或
python scripts/train_all.py

# 只训练部分实验
python scripts/train_all.py --experiments e0,e1

# 单独训练某一组
python scripts/train.py --config configs/e0_rgb_baseline.yaml
python scripts/train.py --config configs/e1_fixed_fusion.yaml
python scripts/train.py --config configs/e2_qaf.yaml
python scripts/train.py --config configs/e3_qaf_p2.yaml

# 从 checkpoint 恢复
python scripts/train.py --config configs/e2_qaf.yaml --resume outputs/e2_qaf/last.pt
```

### 7.3 评估与可视化

```bash
# 单独评估 best 权重
python scripts/eval.py --config configs/e2_qaf.yaml --checkpoint outputs/e2_qaf/best.pt

# 绘制四张 loss 折线图（默认验证集）
python scripts/plot_training.py --split val

# 可选：训练集 loss / 同时绘制 train+val
python scripts/plot_training.py --split train
python scripts/plot_training.py --split both
```

- `loss_total.png`、`loss_box.png`、`loss_cls.png`、`loss_dfl.png`
- `loss_all.png`（2×2 汇总）


## 8. 输出目录约定

每个实验目录（如 `outputs/e2_qaf/`）应包含：

| 文件 | 说明 |
|---|---|
| `config.yaml` | 训练所用配置副本 |
| `run_metadata.json` | git commit、硬件信息、启动时间 |
| `history.json` | 每 epoch 的 train/val loss、AP、lr、耗时、峰值显存 |
| `training_summary.json` | 汇总：best epoch、best AP、总时长、参数量 |
| `best.pt` | 验证 AP 最高 checkpoint |
| `last.pt` | 最后一轮 checkpoint |

### 8.1 `history.json` 单条记录示例

```json
{
  "epoch": 1,
  "train": { "total": 5.07, "box": 1.78, "cls": 1.09, "dfl": 2.20 },
  "val": {
    "total": 3.89, "box": 1.42, "cls": 0.66, "dfl": 1.81,
    "ap": 0.208, "ap50": 0.335, "ap75": 0.237,
    "ap_small": 0.074, "ap_medium": 0.216, "ap_large": 0.199
  },
  "lr": 0.000667,
  "epoch_time_s": 343.18,
  "peak_gpu_mem_mb": 4660.7
}
```

### 8.2 训练策略说明

- **Best 选取**：默认按 **val.ap** 保存 `best.pt`（`train.eval_ap: true`）
- **早停**：`early_stop_patience: 10` 表示 val.ap 连续 10 个 epoch 无提升（且已超过 `early_stop_min_epochs: 20`）则停止
- **最小提升阈值**：`early_stop_min_delta: 0.0001`

## 9. 实验配置要点

四组 YAML 除 `experiment` 与 `output.dir` 外，训练超参 intentionally 保持一致以控制变量：

```yaml
train:
  epochs: 50
  batch_size: 8
  lr: 0.001
  weight_decay: 0.0005
  warmup_epochs: 3
  amp: true
  seed: 42
  eval_ap: true
  early_stop_patience: 10
  early_stop_min_epochs: 20
```

模型规模统一：`width_multiple: 0.5`，`depth_multiple: 0.33`，`max_channels: 512`，`reg_max: 16`，`num_classes: 5`。

|---|---|
